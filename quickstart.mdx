There are three good starting points. Pick the one that matches your goal.

## 1. Just want to try it

<Card title="Open chat.yambr.com" icon="comments" href="https://chat.yambr.com/">
  Hosted Open WebUI with Computer Use already installed and wired up. Sign in, pick a model, ask the assistant to *"create a PowerPoint about Q1 sales"* — you'll see skills, the live browser, and file previews on a real example.
</Card>

Good for: one-off experiments, demos, seeing what the system can do before committing.

## 2. Drive it from your own app (Yambr-hosted)

Use Yambr as the backend — OpenAI-compatible API, no Docker on your side.

<Steps>
  <Step title="Get an API key">
    Sign in at [app.yambr.com](https://app.yambr.com/) and create a key. This is a standard OpenAI-compatible key — it works with the official `openai` SDK, `anthropic` SDK (via LiteLLM), or plain cURL. See [API keys](/platform/api-keys).
  </Step>
  <Step title="Point your client at api.yambr.com">
    ```python
    from openai import OpenAI

    client = OpenAI(
        api_key="sk-yambr-...",              # from app.yambr.com
        base_url="https://api.yambr.com",    # LiteLLM gateway
    )

    resp = client.chat.completions.create(
        model="claude-sonnet-4-6",            # or any model enabled on your key
        messages=[{"role": "user", "content": "Build me a landing page for a coffee shop"}],
        tools=[...],                          # Computer Use tools are routed via cu.yambr.com
    )
    ```

    Full walkthrough: [LiteLLM gateway](/platform/litellm).
  </Step>
  <Step title="Preview artifacts on cu.yambr.com">
    Files the model creates (PDFs, screenshots, HTML) are served from `https://cu.yambr.com/files/{chat_id}/...` and can be embedded as iframes. Artifacts are reachable **only through the LiteLLM gateway** using your key — there is no direct public MCP endpoint. See [cu.yambr.com](/platform/cu-endpoint).
  </Step>
</Steps>

## 3. Self-host everything

Run the whole stack on your own machine. You'll need Docker.

```bash
git clone https://github.com/Yambr/open-computer-use.git
cd open-computer-use
cp .env.example .env
# Edit .env — at minimum set OPENAI_API_KEY (any OpenAI-compatible provider)

# 1. Start the Computer Use Server (builds the workspace image on first run, ~15 min)
docker compose up --build

# 2. In another terminal, start Open WebUI
docker compose -f docker-compose.webui.yml up --build
```

Open [http://localhost:3000](http://localhost:3000). After adding a model in Open WebUI, flip **Function Calling = Native** and **Stream Chat Response = On** — without those two, tools don't fire. Details: [Model settings](/install/model-settings).

Full install guide: [Self-hosting](/install/quickstart) · [Configuration](/install/configuration) · [Docker details](/install/docker).
